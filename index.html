Introduction

This project is a lightweight, concurrent background job manager written in Go. It allows you to submit tasks to execute asynchronously in the background, monitor their execution status, and safely terminate running processes using a local SQLite instance for state persistence.

Project Roadmap

Stack

  • Go: Core language, utilizes os/exec for process control and Go-routines.
  • Sqlite (go-sqlite3): Embedded relational database to track and audit job life cycles.

Job Table Schema

Database table name: job inside local SQLite db (jobs.db).

Column Name Type Description
ID INTEGER (Primary Key) A randomized unique identifier.
PROCESS_ID INTEGER The OS Process ID (PID) of the executed job.
NAME TEXT Human-readable label for the job.
COMMAND TEXT The actual bash script/command executed.
CREATED_AT TEXT Timestamp when the job started (formatted as YYYY-MM-DD HH:MM:SS).
FINNISHED_AT TEXT Timestamp of completion, or "RUNNING" if active.

Usage

Run the manager directly with Go CLI flags. Supported options:

1. Add a Background Job

Spawns a command detached from terminal output and registers it with a random ID:

go run main.go -add_job "sleep 10" -job_name "MySleepJob"

2. List All Jobs

Prints an active, formatted audit trail of all registered tasks:

go run main.go -list

3. Cancel a Job

Sends a SIGKILL signal directly to the underlying system process ID and marks it as completed:

go run main.go -cancel <job_id>
